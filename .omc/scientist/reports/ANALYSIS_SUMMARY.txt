LIMITATIONS
===========

[LIMITATION 1] HTML/CSS Coupling
The footer uses inline styles which cannot be overridden by external media queries
in the same way as class-based CSS. This creates a maintenance issue where
responsive behavior is split across multiple locations (CSS file and HTML style tag).

[LIMITATION 2] Selector Specificity
Inline styles have higher specificity than any external CSS rule. The media query
rule `.ftco-footer .col-md-12 > div` may not match the actual structure,
making it ineffective even if written correctly.

[LIMITATION 3] Gap and Padding Not Adaptive
The 3rem (48px) gap and footer padding are fixed values. On screens narrower than
320px (common for older mobile devices), this reduces available space significantly.

[LIMITATION 4] Logo Sizing Strategy
While logo height is reduced from 240px to 150px on mobile, the flex container
still uses horizontal flow. A better strategy would be to change flex-direction
to column AND adjust the gap/padding accordingly.

[LIMITATION 5] No Mobile-First Approach
The CSS appears to be desktop-first (large default padding 7em) with limited
mobile overrides. A mobile-first approach would make responsive design cleaner.

RECOMMENDATIONS
===============

To fix footer wrapping on mobile, implement:

1. Add proper media query for main footer flex wrapper:
   @media (max-width: 768px) {
     [selector-for-main-flex-wrapper] {
       flex-direction: column;
       gap: 1rem;
       align-items: stretch;
     }
   }

2. Convert inline styles to CSS classes:
   - Move display: flex; from inline style to .footer-content class
   - Create .footer-content-mobile with flex-direction: column

3. Adjust gap and padding responsively:
   - Desktop: gap: 3rem, padding: 2rem 0
   - Mobile: gap: 1rem, padding: 1.5rem 0

4. Ensure logo constraints work with new layout:
   - Use max-width instead of just max-height
   - Test on actual mobile devices (320px, 480px widths)

CONCLUSION
==========

The footer wrapping issue on mobile is caused by:

1. Missing responsive media query for main footer flex wrapper
2. Inline flexbox styles that don't adapt to screen width
3. CSS selector mismatch preventing existing media queries from working
4. Fixed gap (3rem) that doesn't scale to mobile widths
5. Layout design that prioritizes desktop appearance over mobile responsiveness

The website lacks a comprehensive responsive strategy for the footer section.
While some mobile adjustments exist (logo resizing), they don't address the
core layout issue - the flex container needs to change from row to column
on mobile devices.
